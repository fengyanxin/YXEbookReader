# EPUB/PDF 解析功能 - 快速测试指南

## 如何测试

### 步骤 1: 准备测试文件

**推荐使用的测试文件：**

1. **EPUB 文件**：
   - 从 https://www.gutenberg.org/ 下载公版 EPUB
   - 或使用任何标准 EPUB 电子书

2. **PDF 文件**：
   - 使用 Word 导出的 PDF
   - 或任何可以选择文字的 PDF

**不推荐：**
- ❌ 扫描的 PDF/图片
- ❌ 加密的文件
- ❌ 超大文件（>50MB）

### 步骤 2: 上传文件

1. 打开微信开发者工具
2. 点击"上传"标签
3. 点击上传区域
4. 选择"从聊天中选择"
5. 选择你的 EPUB 或 PDF 文件
6. 等待处理（1-10秒）

### 步骤 3: 查看结果

**成功的标志：**
- 显示"解析成功"提示
- 书籍信息被正确提取（标题、作者）
- 可以看到内容预览

**失败的标志：**
- 显示"解析失败"或"无法提取文本"
- 内容为空或只有提示信息

### 步骤 4: 测试阅读

1. 保存书籍到书架
2. 点击书架中的书籍封面
3. 查看是否能正常显示内容

## 调试方法

### 查看日志

1. 打开**调试器**面板
2. 切换到 **Console** 标签
3. 上传文件时观察日志

### 关键日志

**EPUB 解析：**
```
[EPUB] 开始解析
[EPUB] 解压成功
[EPUB] 内容提取成功，长度: 1234
```

**PDF 解析：**
```
[PDF] 开始解析
[PDF] 文件读取成功
[PDF] 找到文本流: 5
[PDF] 文本提取成功，长度: 2345
```

### 常见错误

| 错误 | 原因 | 解决方案 |
|------|------|---------|
| 文件过大 | > 50MB | 使用较小的文件 |
| 不是有效的 EPUB/PDF | 格式错误 | 检查文件是否完整 |
| 无法提取文本 | 图片型/扫描型 | 使用文本型文件 |
| 解压失败 | 文件损坏 | 重新下载文件 |

## 报告问题时请提供

1. 文件格式和大小
2. Console 日志截图
3. 错误提示内容
4. 文件来源

## 功能限制

### EPUB
- ✅ 解压并读取内容
- ✅ 提取标题和作者
- ✅ 提取纯文本
- ❌ 不显示图片
- ❌ 不支持加密文件

### PDF
- ✅ 提取文本内容
- ✅ 提取元数据
- ❌ 图片型 PDF 无法提取
- ❌ 扫描件需要 OCR

## 建议

对于无法解析的文件：
1. 尝试转换为 TXT 格式
2. 使用专门的转换工具
3. 确保文件未加密且包含可提取的文本
